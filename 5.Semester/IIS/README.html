Fituška-v2: sdílení a řešení testových otázek studenty

Rozdělení a popis souborů a složek

./doc: Dokumentace k projektu

./sql: SQL skript pro inicializaci tabulek databáze

./static: Kaskádové styly (css) a používané obrázky

./templates: Vzory pro generování HTML stránek s daty získanými z backendu

./templates/security: HTML vzory pro přihlášování a registrování uživatelů

./src: Pomocné python moduly importované do hlavního programu

./app.py: Hlavní kód aplikace zajišťující vykreslování stránek a back-end k nim

./README.html: Tento dokument

./requirements.txt: Požadované moduly pro fungování aplikace; k jejich stažení lze použít program pip



Lokální instalace aplikace

Pro zprovoznění aplikace je nutné mít nainstalovaný python3 (nejlépe verze 3.7 a výše, u starších verzí není kompatibilita zaručena) ,nástroj pip3 (není nutné) a aplikaci Flask. Jednotlivé moduly programem vyžadované je možné nainstalovat například zmíněným programem pip3 a nebo stáhnout ručně. Vyžadované moduly jsou vypsané v souboru requirements.txt. Pro snadnou instalaci skrz pip je možné použít následující příkaz: pip3 install -r requirements.txt

Lokální instalace databáze

Databáze samotná běží na externím serveru a program se na ni akorát napojuje. V případě zájmu o lokální běh databáze je nutné si zprovoznit mysql server, v něm vytvořit databázi iis (A nebo jinou, potom je ale nutno změnit název i v kódu) a změnit v kódu adresu pro připojení na Vaši lokální. Pro její správnou funkcionalitu je tuto databázi nutno inicializovat použitím skriptu sql_init.sql umístěného ve složce sql tohoto projektu. Tento skript vytvoří tabulky, nastaví správně jejich kódování (pro podporu diakritiky) a naplní je ukázkovými daty.

Spuštění aplikace

Nejjednodušší možnost spuštění aplikace je v kořenovém adresáři napsat příkaz flask run, který implicitně spouští program app.py. Aplikace je poté spuštěna na lokální adrese (localhost či př. 127.0.0.1) na portu 5000. V případě potřeby pro přístupnost serveru z internetu je pak nutno mít otevřený konkrétní port (př. již zmíněný port 5000) a spouštět aplikaci s parametrem --host 0.0.0.0.
